## ✨ О проекте

**Topgram** — это полноценный веб-мессенджер, написанный на Django. Проект демонстрирует глубокое понимание Django ORM, кастомной валидации, шифрования данных и пользовательского опыта. Это не «тудушка» — это готовый к деплою сервис для общения.

### Основные возможности:
- 🔐 **Шифрование сообщений** — все переписки хранятся в зашифрованном виде (cryptography.fernet).
- 👤 **Полноценная система пользователей** — регистрация, авторизация, настройка профиля (аватар, отображаемое имя).
- 💬 **Реальное общение** — отправка сообщений, эмодзи, удаление, статус «онлайн/был недавно».
- 🗂️ **Управление чатами** — чат-лист с последними сообщениями, удаление диалогов, поиск пользователей.
- 🎨 **Современный UI** — тёмная тема, адаптивный дизайн, анимированная страница 404 с космонавтом.
- ⚙️ **Кастомная валидация** — декораторы для проверки логина/пароля (длина, символы, уникальность).
- 🧑‍🔧 **Умная админ-панель** — кастомизированная админка для управления пользователями.

## 🚀 Быстрый старт

### Предварительные требования
- Python 3.12+ установлен
- PostgreSQL или MySQL (опционально, можно с SQLite для теста)

### Установка и запуск

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/kite-house/topgram.git
   cd topgram
   ```

2. **Создайте и активируйте виртуальное окружение**
   ```bash
   python -m venv venv
   source venv/bin/activate  # или venv\Scripts\activate на Windows
   ```

3. **Установите зависимости**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройте переменные окружения**
   
   Скопируйте файл с примером конфигурации:
   ```bash
   cp .env.example .env
   ```
   
   Минимально необходимые настройки:
   ```ini
   # ==================================================
   # ⚙️ REQUIRED ENVIRONMENT VARIABLES
   # ==================================================
   
   # 🚀 Django Secret Key (обязательно сгенерируйте свой)
   SECRET_KEY=ваш_секретный_ключ_django
   
   # 🔐 Fernet Key для шифрования сообщений
   FERNET_SECRET_KEY=ваш_ключ_fernet
   ```

   **Как сгенерировать `FERNET_SECRET_KEY`:**
   ```bash
   python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
   ```

5. **Примените миграции**
   ```bash
   python manage.py migrate
   ```

6. **Создайте суперпользователя**
   ```bash
   python manage.py createsuperuser
   ```

7. **Запустите сервер разработки**
   ```bash
   python manage.py runserver
   ```

8. **Проверьте работу**
   - Главная страница (после входа): http://localhost:8000/
   - Админ-панель: http://localhost:8000/admin/

## 📖 Использование

### 🌐 Регистрация и вход
1. Перейдите на `/accounts/registration` для создания нового аккаунта
2. Войдите на `/login` с вашим username и паролем

### 💬 Общение
- **Начать чат** — воспользуйтесь поиском (поле «Поиск...») и найдите другого пользователя
- **Отправить сообщение** — введите текст в нижнем поле и нажмите Enter или кнопку отправки
- **Эмодзи** — нажмите на иконку смайлика рядом с полем ввода
- **Удалить сообщение** — наведите на сообщение, появится ❌, нажмите на него
- **Удалить чат** — наведите на диалог в левой панели, нажмите ❌

### ⚙️ Настройка профиля
- Нажмите на аватар в правом верхнем углу
- Измените ссылку на аватар или отображаемое имя

### 🎨 Тёмная тема
- Нажмите на иконку луны/солнца в правом верхнем углу для переключения темы

## 🔍 Ключевые технические решения

### 1. Кастомная валидация через декораторы
Класс `Valid` с декоратором `@Valid.registration` проверяет:
- Длину логина (4–30 символов)
- Длину пароля (минимум 5 символов)
- Отсутствие запрещённых символов
- Уникальность пользователя

### 2. Шифрование сообщений
Все сообщения шифруются с помощью `cryptography.fernet` перед сохранением в БД. Ключ Fernet хранится в `.env` и никогда не попадает в код.

### 3. JSONField для хранения структурированных данных
Модель `User` расширена через `AbstractUser`, добавлены поля:
- `chat_list` — список чатов с последними сообщениями
- `messages` — словарь переписки с каждым пользователем

### 4. Динамическое управление статусом «онлайн»
Поле `last_online` обновляется при каждом действии пользователя. Статус считается онлайн, если последняя активность была менее 7 минут назад.

## 📁 Структура проекта

```
Topgram/
├── Oauth2/                         # Приложение авторизации
│   ├── admin.py                    # Кастомизированная админка
│   ├── inspector.py                # Класс Valid с декораторами
│   ├── models.py                   # Расширенная модель User (AbstractUser)
│   ├── views.py                    # Логика регистрации/авторизации
│   └── templates/                  # HTML-шаблоны для логина/регистрации
├── home/                           # Основное приложение чата

│   ├── views.py                    # Главная страница, отправка сообщений, удаление
│   └── templates/home.html         # Интерфейс чата (тёмная тема, эмодзи)
├── error_handler/                  # Кастомные страницы ошибок
│   ├── views.py                    # 404, 500
│   └── templates/error.html        # Анимированная страница с астронавтом
├── topgram/                        # Конфигурация проекта
│   ├── settings.py
│   └── urls.py
├── static/                         # CSS, JS, изображения, favicon
├── .env.example                    # Пример переменных окружения
├── requirements.txt
└── manage.py
```
