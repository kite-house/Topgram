# 🚀 Topgram

### «Асинхронный чат-мессенджер с шифрованием, где даже 404 страница встречает вас лунным астронавтом»

**Topgram** — это не просто учебный проект. Это полноценный, production-ready веб-мессенджер, написанный на Django. Он позволяет пользователям общаться в реальном времени, управлять чатами, настраивать профили и уверенно обрабатывать ошибки. Проект демонстрирует глубокое понимание Django ORM, кастомной валидации, безопасности данных и пользовательского опыта.

---

## ✨ Ключевые особенности

*   **🔐 Безопасное общение:** Все сообщения шифруются с помощью `cryptography.fernet` перед сохранением в базу данных.
*   **👤 Полноценная система пользователей:** Регистрация и авторизация с мощной серверной валидацией (длина логина, допустимые символы, уникальность).
*   **💬 Интуитивный интерфейс чата:**
    *   Отправка и получение сообщений с автоматическим обновлением страницы (`<form>`).
    *   Поддержка эмодзи через удобное всплывающее меню.
    *   Отображение статуса "В сети"/"Был(а) недавно".
    *   Личный чат "Избранное".
*   **🗂️ Управление чатами и сообщениями:**
    *   Контекстное меню для удаления отдельных сообщений или целых чатов.
    *   Чат-лист с сортировкой по последнему сообщению.
    *   Поиск пользователей для начала диалога.
*   **⚙️ Настройка профиля:** Пользователи могут менять свое отображаемое имя и аватар (по URL).
*   **🎨 Продвинутый UI/UX:**
    *   Современный, адаптивный интерфейс с анимациями.
    *   **Переключение темы (Светлая/Темная)** с сохранением в `localStorage`.
    *   **Кастомная страница 404** с космической тематикой (анимация на чистом CSS/JS).
*   **🧑‍🔧 Административная панель:** Кастомизированная админка для управления пользователями.

## 🚀 Быстрый старт

### Предварительные требования

*   Python 3.12+
*   Установленные зависимости из `requirements.txt`

### Установка и запуск

1.  **Клонируйте репозиторий**

2.  **Создайте и активируйте виртуальное окружение**

3.  **Установите зависимости**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Примените миграции**
    ```bash
    python manage.py migrate
    ```

5.  **Создайте суперпользователя**
    ```bash
    python manage.py createsuperuser
    ```

6.  **Настройте переменные окружения** (создайте `.env` файл в корне проекта)
    ```env
    SECRET_KEY=ваш_секретный_ключ_django
    FERNET_SECRET_KEY=ваш_ключ_fernet
    ```

    *   `FERNET_SECRET_KEY` можно сгенерировать в Python:
    ```python
    from cryptography.fernet import Fernet
    print(Fernet.generate_key().decode())
    ```

7.  **Запустите сервер разработки**
    ```bash
    python manage.py runserver
    ```

## 🔍 Глубокий анализ кода (Для технических интервью)

Этот проект показывает, что вы умеете:

1.  **Создавать кастомную бизнес-логику:** Класс `Valid` с декоратором `registration` и `edit_user` — примеры серверной валидации, которую можно легко переиспользовать.
2.  **Безопасно расширять модель пользователя:** Вы использовали `AbstractUser`, добавив такие поля, как `avatar`, `chat_list` и `messages`, а также методы для работы с реальными данными.
3.  **Шифровать данные на уровне приложения:** `cryptography.fernet` используется прямо в `views` перед сохранением сообщения, что демонстрирует понимание "zero-trust" на уровне базы данных.
4.  **Управлять сложными структурами данных:** Вы эффективно используете `JSONField` для хранения данных чатов и сообщений, а функции вроде `display_time` показывают умение работать с датами во фронтенд-части.

## 🧪 Технические детали

*   **Кастомный декоратор `@Valid.registration`:** Обрабатывает всю проверку данных перед созданием нового пользователя.
*   **Шифрование:** Сообщения и логи хранятся в зашифрованном виде, ключ Fernet загружается из `.env`.
*   **Управление состоянием чата:** Класс `UserData` консолидирует всю логику для Главной страницы и настройки профиля.
*   `save_timezone`: Пользовательская сессия для хранения часового пояса, что особенно полезно для интернационализации.
